### 1.2 Agent执行流程

#### ReAct模式执行流程

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                 ReAct Agent 执行流程                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────┐
    │   开始执行   │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │ 1. THINKING  │ ◀── 分析用户指令和当前文档状态
    │   分析思考   │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │ 2. ACTION    │ ◀── 选择并执行适当的工具
    │   执行动作   │     (read_document, edit_document, search_content等)
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │3. OBSERVATION│ ◀── 分析动作执行结果
    │   观察结果   │
    └──────┬───────┘
           │
           ▼
    ┌─────────────────────────────────────────────────────────────┐
    │                    判断是否完成                               │
    │  ┌─────────────────────────────────────────────────────┐   │
    │  │ 是: 完成任务，返回最终结果                          │   │
    │  │ 否: 返回第1步继续循环                               │   │
    │  └─────────────────────────────────────────────────────┘   │
    └─────────────────────────────────────────────────────────────┘
```

#### Planning模式执行流程

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                               Planning Agent 执行流程                                     │

    ┌──────────────┐
    │   开始执行   │
    └──────┬───────┘
           │
           ▼

    └──────┬───────┘
           │
           ▼

    │2. EXECUTION  │ ◀── 执行计划中的当前步骤

    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │3. VALIDATION │ ◀── 验证步骤执行结果
    │   验证结果   │
    └──────┬───────┘
           │
           ▼
    ┌─────────────────────────────────────────────────────────────┐
    │                    判断计划是否完成                          │
    │  ┌─────────────────────────────────────────────────────┐   │
    │  │ 步骤未完成: 继续执行下一个步骤                       │   │
    │  │ 计划已完成: 进入完成阶段                            │   │
    │  └─────────────────────────────────────────────────────┘   │
    └─────────────────────────────────────────────────────────────┘
```

## 三、关键类说明

### 3.1 BaseAgent 核心方法

| 方法 | 描述 |
|------|------|
| `execute()` | 主执行方法，初始化并启动Agent循环 |
| `executeLoop()` | 执行循环，处理每个推理步骤 |
| `buildExecutionPrompt()` | 构建发送给AI模型的提示词 |
| `determineStepType()` | 判断当前步骤类型 |
| `extractThought()` | 从AI响应中提取思考过程 |
| `extractAction()` | 从AI响应中提取执行动作 |
| `extractObservation()` | 从AI响应中提取观察结果 |
| `extractResult()` | 从AI响应中提取最终结果 |
| `isTaskComplete()` | 判断任务是否完成 |

### 3.2 WebSocketService 核心方法

| 方法 | 描述 |
|------|------|
| `registerSession()` | 注册新的WebSocket会话 |
| `unregisterSession()` | 注销WebSocket会话 |
| `bindTaskToSession()` | 将任务ID绑定到会话 |
| `sendToSession()` | 向指定会话发送消息 |
| `broadcast()` | 广播消息到所有会话 |

### 3.3 DocumentService 核心方法

| 方法 | 描述 |
|------|------|
| `createDocument()` | 创建新文档 |
| `getDocument()` | 获取文档 |
| `updateDocument()` | 更新文档内容 |
| `executeAgentTask()` | 执行Agent任务 |
| `generateDiff()` | 生成Diff对比 |
