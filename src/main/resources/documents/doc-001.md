## 简介

### 什么是 LangChain？

LangChain 是一个用于开发由**大语言模型（LLM）驱动的应用程序**的开源框架。它由 Harrison Chase 于 2022 年底创建，迅速成为 LLM 应用开发领域最受欢迎的工具之一。

LangChain 的核心设计理念是**组件化**和**可组合性**。它提供了一系列标准化的组件，开发者可以像搭积木一样自由组合这些组件，快速构建复杂的 LLM 应用。

### 核心价值主张

LangChain 解决了 LLM 应用开发中的几个关键问题：

1. **模型抽象** - 统一不同 LLM 提供商的 API 接口，一套代码可以轻松切换模型
2. **提示管理** - 提供模板化的提示管理系统，支持变量注入和动态生成
3. **上下文管理** - 内置记忆系统，支持多轮对话和长文本处理
4. **工具集成** - 让 LLM 能够调用外部工具和 API，扩展能力边界
5. **流程编排** - 支持复杂的链式调用和智能体工作流

### 为什么选择 LangChain？

| 特性 | 描述 | 适用场景 |
|------|------|----------|
| 统一接口 | 同一套代码切换不同 LLM | 多模型对比测试、成本优化 |
| 记忆系统 | 内置对话记忆管理 | 聊天机器人、客服系统 |
| 工具调用 | LLM 可调用外部工具 | Agent 应用、自动化任务 |
| 链式调用 | 多步骤任务编排 | 复杂工作流、数据处理 |
| RAG 支持 | 原生支持检索增强生成 | 知识库问答、文档分析 |
| 流式输出 | 支持 Token 级别流式返回 | 提升用户体验、实时响应 |

### LangChain 生态系统

LangChain 不仅仅是一个库，而是一个完整的生态系统：

- **LangChain** - 核心框架，提供组件和抽象
- **LangGraph** - 构建状态化、多角色应用的框架
- **LangSmith** - 应用调试、测试和监控平台
- **LangServe** - 将链部署为 REST API 的工具

## 核心概念

LangChain 的架构建立在几个核心概念之上，理解这些概念是掌握 LangChain 的关键。

### 3.1 模型

模型是 LangChain 的基础组件，负责与 LLM 进行交互。LangChain 支持两种主要的模型类型：

#### 3.1.1 LLM（大语言模型）

LLM 是传统的文本输入-文本输出模型，适合简单的文本生成任务：

```python
from langchain_openai import OpenAI

# 初始化 LLM
llm = OpenAI(
    model="gpt-3.5-turbo-instruct",
    temperature=0.7,
    max_tokens=256
)

# 简单调用
response = llm.invoke("什么是机器学习？请用一句话解释。")
print(response)

# 批量调用
responses = llm.batch([
    "什么是深度学习？",
    "什么是自然语言处理？",
    "什么是计算机视觉？"
])
for r in responses:
    print(r)
    print("---")
```

#### 3.1.2 Chat Models（聊天模型）

Chat Models 是更现代的接口，支持多角色消息和多轮对话：

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import (
    HumanMessage,
    SystemMessage,
    AIMessage
)

# 初始化聊天模型
chat = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0
)

# 构建消息列表
messages = [
    SystemMessage(content="你是一个专业的 Python 开发者，擅长简洁清晰地解释技术概念。"),
    HumanMessage(content="请解释 Python 中的装饰器是什么？"),
]

# 调用模型
response = chat.invoke(messages)
print(response.content)

# 继续对话
messages.append(response)  # 添加 AI 的回复
messages.append(HumanMessage(content="能给我一个实际例子吗？"))

# 第二轮对话
response2 = chat.invoke(messages)
print(response2.content)
```

#### 3.1.3 模型参数详解

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    # 模型名称
    model="gpt-4o-mini",
    
    # 温度参数（0-2）：控制随机性
    # 0 - 最确定性，适合代码生成、事实性问题
    # 0.7 - 平衡创造性和一致性，适合一般对话
    # 1.5+ - 高随机性，适合创意写作
    temperature=0.7,
    
    # 最大 Token 数
    max_tokens=1024,
    
    # Top-p 采样（0-1）
    top_p=0.9,
    
    # 频率惩罚（-2 到 2）
    # 正值降低重复词语出现的概率
    frequency_penalty=0.5,
    
    # 存在惩罚（-2 到 2）
    # 正值鼓励讨论新话题
    presence_penalty=0.5,
    
    # 停止词
    stop=["\n\n", "END"],
    
    # 请求超时（秒）
    timeout=30,
    
    # 最大重试次数
    max_retries=2,
)
```

### 3.2 提示模板

提示模板是 LangChain 的核心组件之一，用于动态生成发送给 LLM 的提示。

#### 3.2.1 基础提示模板

```python
from langchain_core.prompts import PromptTemplate

# 方式一：from_template（自动推断变量）
template = PromptTemplate.from_template(
    "请用{style}的风格，解释{concept}这个概念。"
)

# 渲染模板
prompt = template.invoke({
    "style": "通俗易懂",
    "concept": "递归"
})
print(prompt.text)
# 输出：请用通俗易懂的风格，解释递归这个概念。

# 方式二：显式定义变量
template = PromptTemplate(
    template="请用{style}的风格，解释{concept}这个概念。",
    input_variables=["style", "concept"]
)

# 验证变量
print(template.input_variables)  # ['style', 'concept']
```

### 3.3 输出解析器

输出解析器用于将 LLM 的文本输出转换为结构化数据。

#### 3.3.1 字符串解析器

```python
from langchain_core.output_parsers import StrOutputParser

# 最简单的解析器，直接返回字符串
parser = StrOutputParser()

result = parser.parse("Hello, World!")
print(result)  # Hello, World!
```

#### 3.3.2 JSON 解析器

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.pydantic_v1 import BaseModel, Field
from typing import List

# 定义数据模型
class Person(BaseModel):
    name: str = Field(description="人物姓名")
    age: int = Field(description="人物年龄，必须是正整数")
    occupation: str = Field(description="职业")
    skills: List[str] = Field(description="技能列表")
    email: str = Field(description="邮箱地址")

# 创建解析器
parser = JsonOutputParser(pydantic_object=Person)

# 获取格式说明（可插入到提示中）
format_instructions = parser.get_format_instructions()
print(format_instructions)
# 输出类似：
# The output should be formatted as a JSON instance that conforms to the JSON schema below.
# ...
```

#### 3.3.3 列表解析器

```python
from langchain_core.output_parsers import CommaSeparatedListOutputParser

# 解析逗号分隔的列表
parser = CommaSeparatedListOutputParser()

# 获取格式说明
format_instructions = parser.get_format_instructions()

# 解析输出

print(result)  # ['苹果', '香蕉', '橘子', '葡萄']
```

### 3.4 链

链是 LangChain 中组合多个组件的核心机制。

#### 3.4.1 LCEL（LangChain Expression Language）

LCEL 是 LangChain 的表达式语言，使用 `|` 操作符连接组件：

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 定义组件
model = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_template("给我讲一个关于{topic}的笑话")
parser = StrOutputParser()

# 使用 LCEL 组合
chain = prompt | model | parser

# 调用
result = chain.invoke({"topic": "程序员"})
print(result)
```

#### 3.4.2 LCEL 的优势

| 特性 | 说明 |
|------|------|
| 流式输出 | 自动支持流式返回 |
| 异步支持 | 同步和异步 API 一致 |
| 批处理 | 自动支持批量调用 |
| 并行执行 | 自动并行化独立操作 |
| 重试和回退 | 内置错误处理机制 |

#### 3.4.3 Runnable 接口

LCEL 的核心是 Runnable 接口，所有组件都实现了这个接口：

```python
# invoke - 单次调用
result = chain.invoke({"topic": "Python"})

# batch - 批量调用
results = chain.batch([
    {"topic": "Python"},
    {"topic": "Java"},
    {"topic": "JavaScript"}
])

# stream - 流式输出
for chunk in chain.stream({"topic": "Python"}):
    print(chunk, end="", flush=True)

# ainvoke - 异步调用
import asyncio
result = await chain.ainvoke({"topic": "Python"})

# abatch - 异步批量
results = await chain.abatch([...])

# astream - 异步流式
async for chunk in chain.astream({"topic": "Python"}):
    print(chunk, end="", flush=True)
```

## 快速开始

### 你的第一个 LangChain 应用

让我们创建一个简单但完整的问答系统：

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 第一步：创建模型
model = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7
)

# 第二步：创建提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个友好的助手，擅长用简单易懂的方式回答问题。"),
    ("human", "{question}")
])

# 第三步：创建输出解析器
parser = StrOutputParser()

# 第四步：组合成链
chain = prompt | model | parser

# 第五步：调用链
while True:
    question = input("\n请输入问题（输入 'quit' 退出）: ")
    if question.lower() == 'quit':
        break
    
    answer = chain.invoke({"question": question})
    print(f"\n回答: {answer}")
```

### 流式输出示例

提升用户体验的关键技术：

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

model = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_template("写一篇关于{topic}的短文")

# 流式输出

```

### 批量处理示例

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

model = ChatOpenAI(model="gpt-4o-mini")
```

### 6.1 文档加载器

LangChain 提供了丰富的文档加载器，支持多种数据源：

#### 文本文件

```python
from langchain_community.document_loaders import TextLoader

# 加载文本文件
loader = TextLoader("./documents/notes.txt", encoding="utf-8")
documents = loader.load()

print(documents[0].page_content[:200])  # 打印前 200 个字符
```

#### PDF 文件

```python
from langchain_community.document_loaders import PyPDFLoader

# 加载 PDF
loader = PyPDFLoader("./documents/report.pdf")
pages = loader.load_and_split()

print(f"共 {len(pages)} 页")

# 每页作为一个文档
for i, page in enumerate(pages):
    print(f"第 {i+1} 页: {len(page.page_content)} 字符")
```

#### 网页

```python
from langchain_community.document_loaders import WebBaseLoader

# 加载网页
loader = WebBaseLoader("https://en.wikipedia.org/wiki/Artificial_intelligence")
documents = loader.load()

print(documents[0].page_content[:500])
```

#### CSV 文件

```python
from langchain_community.document_loaders import CSVLoader

# 加载 CSV
loader = CSVLoader(
    file_path="./data/products.csv",
    csv_args={
        "delimiter": ",",
        "quotechar": '"'
    }
)
documents = loader.load()

# 每行作为一个文档

```

#### 目录批量加载

```python
from langchain_community.document_loaders import DirectoryLoader

# 加载目录下所有文件

    show_progress=True
)
documents = loader.load()

print(f"共加载 {len(documents)} 个文档")
```

### 6.2 文本分割器

长文档需要分割成小块才能有效处理：

```python
from langchain_text_splitters import (
    RecursiveCharacterTextSplitter,
    CharacterTextSplitter,
    TokenTextSplitter
)

text = "这是一段很长的文本..." * 100

# 方式一：递归字符分割器（推荐）
recursive_splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,      # 每块最大字符数
    chunk_overlap=50,    # 块之间的重叠字符数
    length_function=len,  # 长度计算函数
    separators=["\n\n", "\n", "。", "，", " ", ""]  # 分隔符优先级
)

chunks = recursive_splitter.split_text(text)
print(f"分割成 {len(chunks)} 块")

# 方式二：字符分割器
```

### 7.1 检索增强生成（RAG）

RAG 是最流行的 LLM 应用模式之一：

```
┌─────────────────┐
│    用户提问      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐     ┌─────────────────┐
│   问题嵌入向量化  │────▶│    向量数据库    │
└─────────────────┘     └────────┬────────┘
                                 │
                                 ▼ 相似性搜索
                        ┌─────────────────┐
                        │    相关文档片段   │
                        └────────┬────────┘
                                 │
                                 ▼
┌─────────────────┐     ┌─────────────────┐
│    最终答案      │◀────│  LLM 生成回答   │
└─────────────────┘     └────────┬────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │  问题 + 上下文   │
                        │  + 相关文档      │
                        └─────────────────┘
```

#### 完整 RAG 实现

```python

    documents=splits,
    embedding=embeddings,
    persist_directory="./manual_vectorstore"
)

# 4. 创建检索器

    search_type="similarity",
    search_kwargs={"k": 4}
)

# 5. 创建 RAG 链
```

### 案例1：智能文档问答系统

构建一个完整的文档问答系统：

```python
import os
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.document_loaders import PyPDFLoader, TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_chroma import Chroma
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables import RunnableWithMessageHistory

class DocumentQA:
    def __init__(self, pdf_path: str, persist_dir: str = "./chroma_db"):
        self.pdf_path = pdf_path
        self.persist_dir = persist_dir
        self.store = {}
        
        # 初始化模型
        self.model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
        self.embeddings = OpenAIEmbeddings()
        
        # 初始化向量存储
        self.vectorstore = self._init_vectorstore()
        self.retriever = self.vectorstore.as_retriever(
            search_type="similarity",
            search_kwargs={"k": 4}
        )
        
        # 创建链
        self.chain = self._create_chain()
    
    def _init_vectorstore(self):
        """初始化或加载向量存储"""
        if os.path.exists(self.persist_dir):
            print("加载已有向量存储...")
            return Chroma(
                persist_directory=self.persist_dir,
                embedding_function=self.embeddings
            )
        
        print("创建新的向量存储...")
        # 加载文档
        loader = PyPDFLoader(self.pdf_path)
        docs = loader.load()
        
        # 分割
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=1000,
            chunk_overlap=200
        )
        splits = splitter.split_documents(docs)
        
        # 创建向量存储
        return Chroma.from_documents(
            documents=splits,
            embedding=self.embeddings,
            persist_directory=self.persist_dir
        )
    
    def _create_chain(self):
        """创建 RAG 链"""
        prompt = ChatPromptTemplate.from_messages([
            ("system", """你是一个专业的文档问答助手。请基于以下上下文回答问题。
            
上下文：
{context}

要求：
1. 只使用上下文中的信息回答
2. 如果上下文没有相关信息，说明"文档中没有找到相关信息"
3. 引用具体的来源（如页码）
4. 回答要简洁准确
"""),
            ("placeholder", "{chat_history}"),
            ("human", "{input}")
        ])
        
        chain = prompt | self.model
        
        return RunnableWithMessageHistory(
            chain,
            self._get_session_history,
            input_messages_key="input",
            history_messages_key="chat_history"
        )
    
    def _get_session_history(self, session_id: str):
        if session_id not in self.store:
            self.store[session_id] = InMemoryChatMessageHistory()
        return self.store[session_id]
    
    def ask(self, question: str, session_id: str = "default"):
        """提问"""
        # 检索相关文档
        docs = self.retriever.invoke(question)
        context = "\n\n".join(doc.page_content for doc in docs)
        
        # 调用链
        response = self.chain.invoke(
            {"input": question, "context": context},
            config={"configurable": {"session_id": session_id}}
        )
        
        return {
            "answer": response.content,
            "sources": docs
        }

# 使用
qa = DocumentQA("./technical_manual.pdf")

# 多轮对话
print(qa.ask("这个系统的主要功能是什么？")["answer"])
print(qa.ask("如何安装？")["answer"])
print(qa.ask("安装过程中需要注意什么？")["answer"])  # 会记住之前的问题
```

### 错误处理

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnableRetry

model = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_template("{input}")
parser = StrOutputParser()

# 方法一：使用 RunnableRetry
chain_with_retry = (
    prompt 
    | model.with_retry(wait=3, stop_after_attempt=3) 
    | parser
)

# 方法二：自定义错误处理
from langchain_core.runnables import RunnableLambda

def handle_error(error):
    return f"抱歉，处理请求时出错：{str(error)}"

chain_with_fallback = (
    prompt 
    | model 
    | parser
).with_fallbacks([
    RunnableLambda(handle_error)
])

# 方法三：使用 try-except
try:
    result = chain.invoke({"input": "Hello"})
except Exception as e:
    print(f"Error: {e}")
    result = "默认回复"
```

### 常见陷阱

- [x] **API Key 硬编码** - 使用环境变量或密钥管理服务
- [x] **无限 Token 消耗** - 设置 `max_tokens` 和成本监控
- [x] **缺少超时设置** - 所有 API 调用都应设置超时
- [x] **忽略错误处理** - 使用 try-except 和重试机制
- [x] **全局状态污染** - 使用会话隔离，不要用全局变量存储用户数据
- [x] **过度依赖模型** - 能用规则解决的不要用 LLM
